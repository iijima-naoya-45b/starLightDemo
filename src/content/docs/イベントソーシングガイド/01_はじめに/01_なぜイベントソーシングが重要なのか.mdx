# なぜイベントソーシングが重要なのか

イベントソーシングは、状態の変更をイベントとして保存するアーキテクチャパターンです。

## イベントソーシングの重要性

### イベントソーシングなしの問題

**問題のある状況:**

```
従来のアプローチ:
- 現在の状態のみ保存
- 変更履歴が失われる
- 監査が困難
- 過去の状態を再現できない
```

### イベントソーシングによる解決

**改善された状況:**

```
イベントソーシングにより:
- すべての変更が記録される
- 変更履歴が保持される
- 監査が容易
- 過去の状態を再現できる
```

## まとめ

イベントソーシングが重要な理由：

- **監査**: すべての変更が記録される
- **タイムトラベル**: 過去の状態を再現できる
- **デバッグ**: 問題の原因を特定しやすい
- **柔軟性**: 新しい読み取りモデルを追加できる

適切なイベントソーシングにより、監査可能で柔軟なシステムを構築できます。
