## 🧭 羅針盤資料の作成：迷いという「状態異常」を無効化する

**この文書は「ドキュメントという概念を解体し、実行可能な仕組みに置換する」ための宣言書です。**

### 1. 💀 情報の「ひんし」判定

手動で入力されたすべての文字は、入力した瞬間から**「どく」状態に陥り、将来的に嘘をつく「裏切り者」**へと変貌します。

**正義は「自動生成」:**

GitHub ActionsやCLIによる自動生成こそが、情報の鮮度を保つ唯一の**「リカバー」**である。

手動メンテナンスが必要なドキュメントは、書いた瞬間に「負債」という名の**「のろい」**が発動していると自覚せよ。

### 2. ⚡ フロー状態という「せいしんりょく」の死守

「ドキュメントを探すためにブラウザを開く」「ツールを切り替える」行為は、エンジニアの全能力を削る**「いやなおと」**であり、創造性への攻撃です。

**IDE（戦場）からの離脱禁止:**

**「そらをとぶ」で別の画面へ移動する必要はない。**ターミナルやEditor上で完結する設計こそが、エンジニアの**「とくせい：せいしんりょく（怯まない）」**を最大化させる。

検索させるな。今書いているコードのすぐ横に、必要な情報が**「しぜんのめぐみ」**のように湧き出る設計を構築しなさい。

### 3. 🛡️ 信頼の「ずかん」スコアリング

人間が「この情報は信じて良いか」を疑う時間は、脳のリソースを無駄にする**「わるあがき」**です。

**「み破る」をシステム化せよ:**

すべての情報のヘッダーに、自動更新日時とソースコードリンクを強制表示し、情報の出処を**「鑑定」**不要の状態にせよ。

**情報のヘッダー例:**

```
最終自動更新: 2026-02-01 12:00 JST
生成元: ./scripts/generate-portal.ts
参照元: [GitHub: source-code-link]
```

### 4. 🛰️ 地図（Portal）の自動生成：唯一の「タウンマップ」

Portalは「情報の地図」です。**「地図だけは一元化し、内容は分散させる」**という、四天王を倒すための唯一の戦略を採用します。

**自動生成する「わざマシン」の内容:**

- README / Portal（入り口の統一）
- 環境構築コマンド（一撃でセットアップ）
- 重要URL / ADR一覧（意思決定のログ）
- CI/CDルール（戦場の掟）

### 5. 🪓 「情報の死」を防ぐ絶対の禁じ手

- **手動更新は「じばく」と同義:** 更新を忘れた瞬間、チーム全体に大ダメージを与える。
- **自動生成が「せいなるつるぎ」:** 嘘を切り裂き、常に真実だけを照らす。
- **出せない情報は「封印」せよ:** 更新できない情報は、最初から存在しない方が全滅を防げる。

### 🪓 トップトレーナーへの最終助言

**「ドキュメントを1行書く暇があるなら、テストコードか自動化スクリプトを1行書け。それが最強の『きずな』を生む。」**

ドキュメントという概念を解体し、実行可能な仕組みに置換すること。エンジニアが「ルール」を意識せずとも、自動生成された地図に従うだけで最高打点のわざを繰り出せる環境を整えること。それが、この羅針盤資料における唯一にして最大の目的です。
