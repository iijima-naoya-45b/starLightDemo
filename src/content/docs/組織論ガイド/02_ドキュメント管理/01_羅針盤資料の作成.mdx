## 羅針盤資料の作成

**この文書は「ドキュメントという概念を解体し、実行可能な仕組みに置換する」ための宣言書です。**

### 1. 情報の死の定義

手動で入力されたすべての文字は、**将来の嘘**です。  
更新されない瞬間から、情報は「死にます」。  
**正義は自動生成**です。GitHub Actions / CLI による生成をデフォルトとします。

### 2. フロー状態の死守

「資料を読む」「ツールを切り替える」行為は、エンジニアの創造性への**攻撃**です。  
**IDEやターミナルから一歩も出ずに**、すべての情報が完結する設計に置換します。

```
原則:
- 情報はコードから自動生成する
- 必要な情報はCLIやEditor上で参照できる
- 検索ではなく「今いる場所に現れる」設計にする
```

### 3. 信頼のスコアリング

人間が「この情報は信じて良いか」を判断する時間は、無駄です。  
すべての情報のヘッダーに、**自動更新日時とソースコードリンク**を強制表示します。

```
例:
最終自動更新: 2024-06-01 12:00 UTC
生成元: ./scripts/generate-portal.ts
参照コード: https://github.com/org/repo/blob/main/scripts/generate-portal.ts
```

### 4. 実装の基本構成（Portalの自動生成）

Portalは「情報の地図」です。  
**地図だけは一元化し、内容は分散させる**という唯一の正解を採用します。

```
自動生成対象:
- README / Portal
- 環境構築コマンド
- 重要URL
- ADR一覧
- CI/CDルール

生成方法:
- GitHub Actionsで定期生成
- CLIでローカル生成
```

### 5. 「情報の死」を防ぐ唯一の運用

- **手動更新は禁止**  
- **自動生成が唯一の正義**  
- **更新できない情報は最初から出さない**  

### 覚醒の結び

**ドキュメントを1行書く暇があるなら、テストコードか自動化スクリプトを1行書け。**
