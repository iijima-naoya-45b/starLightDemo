## Rails における View の詳細
Rails における View は、ユーザーに表示されるインターフェースを担当します。Controller から渡されたデータを受け取り、HTML、CSS、JavaScript を使ってウェブページを生成します。

### 1. View の役割
View の主な役割は、Controller から受け取ったデータを、ユーザーが理解しやすい視覚的な形式に変換することです。View はデータ表示に特化しており、ビジネスロジックやデータベース操作は含みません。

### 2. View のテンプレートエンジン
Rails は、ERB (Embedded RuBy) というテンプレートエンジンをデフォルトで採用しています。ERB ファイルは、HTML の中に Ruby のコードを埋め込むことができ、`.html.erb` という拡張子を持ちます。

- `<%= ... %>`: このタグは、Ruby のコードを実行し、その結果を HTML に出力します。データの表示によく使われます。

- `<% ... %>`: このタグは、Ruby のコードを実行しますが、その結果は出力されません。ループや条件分岐などの制御構造に使われます。

### 3. View のファイル構成
View ファイルは、対応する Controller 名のディレクトリ内に配置されます。例えば、`UsersController` の View ファイルは `app/views/users/` ディレクトリに置かれます。各アクションに対応する View ファイルは、アクション名と同じ名前になります（例：`index` アクションは `index.html.erb`）。

### 4. レイアウトと部分テンプレート
Rails の View には、コードの再利用性を高めるための仕組みが用意されています。

- **レイアウト (Layout)**:
  `app/views/layouts/` ディレクトリにあるレイアウトファイルは、ウェブサイト全体の共通部分（ヘッダー、フッター、ナビゲーションバーなど）を定義します。各 View はこのレイアウトの中に埋め込まれます。これにより、各ページで同じ HTML を繰り返し書く必要がなくなります。
  `yield` キーワードが、各アクションの View コンテンツを挿入する場所を指定します。

- **部分テンプレート (Partial)**:
  複数の View で共通して使われる HTML の断片（例：フォーム、ユーザーカード）を、`_`（アンダースコア）から始まるファイルとして切り出します。これにより、コードの重複を防ぎ、保守性を向上させます。部分テンプレートは `render` メソッドで呼び出せます。

### 5. まとめ
Rails の View は、ユーザーインターフェースの生成に特化したコンポーネントです。Controller から受け取ったデータを基に、ERB を使って動的な HTML を生成します。レイアウトや部分テンプレートを活用することで、効率的で保守性の高いビュー層を構築できます。
