## 特定商取引法基礎

**⚠️ この資料の致命的な欠陥を指摘します。**

拝読いたしました。貴殿の案は、組織を「ルールに依存する無能な集団」へと導く、非常に危険な思想に基づいています。

「法律」という、不可侵の聖域のような言葉を盾にしながら、その実態は「チェックボックスを増やせば解決する」という、エンジニアリングを軽視した事務作業の推奨に過ぎません。この資料が現場に持ち込まれた場合、半年以内に開発スピードは法務確認という名の「官僚的承認プロセス」によって窒息するでしょう。

特定商取引法（特商法）は、通信販売や電子商取引における消費者保護を目的とした法律です。

しかし、この資料には以下の致命的な欠陥があります。

```
【致命的過失1: 「読解という名の無賃労働」：静的なHTMLへの執着】

事業者名や所在地をHTMLに直書きすることを推奨している点は、
エンジニアリング組織としての知性を疑わざるを得ません。

会社情報の変更や、複数のショップ展開、あるいは法改正による表示項目の追加のたびに、
エンジニアに「HTMLの修正」という低次元なタスクを強いるつもりですか？

なぜこれらの情報を「マスターデータ」として一元管理し、APIやコンポーネントを通じて
一貫性を保つ仕組みを提示しないのでしょうか。

現実には、以下のような状況が発生します：

1. **会社情報の変更による修正漏れ**:
   - 会社情報が変更された場合、すべてのHTMLを修正する必要がある
   - 修正漏れが発生すると、法的リスクが生じる
   - その結果、エンジニアは「HTMLの修正」という低次元なタスクに時間を浪費する

2. **複数のショップ展開による非効率**:
   - 複数のショップを展開する場合、それぞれのHTMLを修正する必要がある
   - 修正漏れが発生すると、法的リスクが生じる
   - その結果、エンジニアは「HTMLの修正」という低次元なタスクに時間を浪費する

3. **法改正による表示項目の追加**:
   - 法改正により表示項目が追加された場合、すべてのHTMLを修正する必要がある
   - 修正漏れが発生すると、法的リスクが生じる
   - その結果、エンジニアは「HTMLの修正」という低次元なタスクに時間を浪費する

その親切心が、いかに現場の首を絞めているかという残酷な真実を暴きます。

事業者名や所在地をHTMLに直書きすることを推奨する姿勢は、
エンジニアリング組織としての知性を疑わざるを得ず、将来の「修正漏れ」という
法的リスクを再生産しているだけです。
```

### 特定商取引法で必要な表示事項

#### 1. 事業者情報

- **事業者名**: 法人の場合は商号、個人の場合は氏名
- **所在地**: 本店または主たる事務所の所在地
- **電話番号**: 問い合わせ先の電話番号
- **代表者名**: 法人の場合は代表者の氏名

#### 2. 商品・サービス情報（注意: 「スケーラビリティの完全な無視」：情報のハードコーディング）

```

1. **プラットフォームによる抽象化**:

```typescript
// コア・エンジンの仕様としての抽象化
class LegalComplianceEngine {
  async getShippingInfo(productId: string, country: string) {
    // 配送ロジックから自動計算
    const shippingLogic = await this.shippingLogicRepository.get(productId);
    const shippingInfo = await shippingLogic.calculate({
      productId,
      country,
      legalRequirements: this.getLegalRequirements(country),
    });
    
    return {
      shippingFee: shippingInfo.fee,
      shippingMethod: shippingInfo.method,
      estimatedDelivery: shippingInfo.estimatedDelivery,
      // 法的に完璧な情報が自動的にインジェクトされる
    };
  }
  
  async getPaymentInfo(productId: string, country: string) {
    // 支払方法を法規制に応じて自動生成
    const legalRequirements = this.getLegalRequirements(country);
    const paymentMethods = await this.paymentMethodRepository.getAvailable({
      productId,
      country,
      legalRequirements,
    });
    
    return {
      methods: paymentMethods,
      paymentTiming: this.getPaymentTiming(country),
      // 法的に完璧な情報が自動的にインジェクトされる
    };
  }
  
  private getLegalRequirements(country: string) {
    // 各国の法規制に応じて、自動的に要件を取得
    return this.legalRequirementsRepository.getByCountry(country);
  }
}

// フロントエンド実装者が意識せずとも、法的に完璧な情報がインジェクトされる
export function ProductPage({ productId }: { productId: string }) {
  const shippingInfo = useShippingInfo(productId); // 自動的に法的に完璧な情報が取得される
  const paymentInfo = usePaymentInfo(productId); // 自動的に法的に完璧な情報が取得される
  
  return (
    <div>
      <ProductDetails />
      <ShippingInfo info={shippingInfo} /> {/* 自動的に法的に完璧な情報が表示される */}
      <PaymentInfo info={paymentInfo} /> {/* 自動的に法的に完璧な情報が表示される */}
    </div>
  );
}
```

# - 商品ごとに手動で記述する必要がない

// チェックアウトフロー全体のバリデーター
class CheckoutValidator {
  validateLegalRequirements(checkoutData: CheckoutData): ValidationResult {
    const errors: string[] = [];
    
    // 最終確認画面の要件チェック
    if (!checkoutData.finalConfirmationScreen) {
      errors.push('最終確認画面が存在しません');
    }
    
    if (!checkoutData.finalConfirmationScreen.purchaseButtonText) {
      errors.push('購入確定ボタンの文言が設定されていません');
    }
    
    if (!checkoutData.finalConfirmationScreen.cancellationTerms) {
      errors.push('定期購入の解約条件が表示されていません');
    }
    
    // 事業者情報のチェック
    if (!checkoutData.sellerInfo.companyName) {
      errors.push('事業者名が表示されていません');
    }
    
    // 返品・交換条件のチェック
    if (!checkoutData.returnPolicy.terms) {
      errors.push('返品・交換条件が表示されていません');
    }
    
    return {
      isValid: errors.length === 0,
      errors,
    };
  }
}
