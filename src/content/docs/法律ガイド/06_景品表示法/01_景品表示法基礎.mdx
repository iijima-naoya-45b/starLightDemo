## 景品表示法基礎

**⚠️ この資料の致命的な欠陥を指摘します。**

拝読いたしました。貴殿の案は、組織を「ルールに依存する無能な集団」へと導く、非常に危険な思想に基づいています。

「景品表示法」という、現代において最もクリティカルな技術課題を、「根拠のある表現にせよ」と日本語で周知し、その判断を個々のエンジニアやライターの「良識」に委ねる行為は、知的リソースの浪費です。この資料が現場に持ち込まれた場合、半年以内に「2年以下の懲役または300万円以下の罰金」という制裁によって組織の存続が危うくなるでしょう。

景品表示法は、不当な景品類及び不当表示を防止する法律です。

### なぜ景品表示法が重要なのか

#### 問題のある広告・表示

**問題のある広告・表示:**

```html
<!-- 誇大広告 -->
<div class="advertisement">
  <h2>世界一の商品！</h2>
  <p>他の商品より100倍効果的！</p>
  <!-- 問題点:
      - 根拠のない誇大表現
      - 比較対象が不明確
      - 証拠がない
  -->
</div>
```

**景品表示法に準拠した広告・表示:**

```html
<!-- 適切な広告・表示 -->
<div class="advertisement">
  <h2>高品質な商品</h2>
  <p>当社のテスト結果によると、一般的な商品と比較して効果が高いことが確認されました。</p>
  <p class="disclaimer">※効果には個人差があります</p>
  <!-- メリット:
      - 根拠のある表現
      - 比較対象が明確
      - 適切な注意書き
  -->
</div>
```

**メリット:**
1. **消費者保護**: 消費者が適切な情報を得られる
2. **法的リスクの回避**: 違反による罰則を回避
3. **信頼性**: 適切な表示で信頼を獲得

### 景品表示法の主要な規制

#### 1. 優良誤認表示

優良誤認表示は、商品やサービスの品質、内容などについて、実際のものよりも著しく優良であると誤認させる表示です。

**禁止される表示（注意: 「読解という名の無賃労働」：定性的な「表現」に依存する危うさ）:**

- **根拠のない「最高」「最良」などの表現**
- **比較対象が不明確な比較広告**
- **証拠のない効果の表示**

**❌ 問題のあるアプローチ（「根拠のある表現にせよ」と日本語で周知し、その判断を個々のエンジニアやライターの「良識」に委ねる）:**

```
# 問題点:
# 1. 「根拠のある表現にせよ」と日本語で周知し、その判断を個々のエンジニアやライターの
#    「良識」に委ねる行為は、知的リソースの浪費
# 2. 「最高」「世界初」といったNGワードの監視を、なぜ人間の目視に頼るのか
# 3. なぜこれを、コンテンツ入稿時の自然言語処理（NLP）による自動検閲や、
#    禁止語句が含まれる場合に公開をブロックするCIパイプラインのバリデータとして
#    実装しないのか
# 4. 人間がガイドラインを読み、人間が修正案を考える……。そんな不確実な「人力フィルター」を
#    周知し合う労力は、将来の「うっかりミス」による措置命令のリスクを何ら低減させない
```

#### 2. 有利誤認表示

有利誤認表示は、商品やサービスの価格、取引条件などについて、実際のものよりも著しく有利であると誤認させる表示です。

**禁止される表示:**

- **虚偽の「限定」「先着」などの表現**
- **実際より高い価格からの割引表示**
- **取引条件の虚偽表示**

#### 3. 実践例: 適切な広告・表示（注意: 「周知という名の責任転嫁」「システムへの信頼の欠如」「スケーラビリティの完全な無視」）

**❌ 問題のあるアプローチ（価格比較や「期間限定」をHTMLに直書き、静的なテキストで周知、情報のハードコーディング）:**

```html
<!-- 適切な広告・表示 -->
<div class="product-ad">
  <h2>新商品発売</h2>
  <p class="price">
    通常価格: <span class="original-price">¥10,000</span>
    特別価格: <span class="sale-price">¥8,000</span>
  </p>
  <p class="disclaimer">
    ※特別価格は2024年1月31日まで有効です
    ※在庫に限りがあります
  </p>
  
  <div class="effect-claim">
    <h3>効果について</h3>
    <p>当社のテスト結果によると、一般的な商品と比較して効果が高いことが確認されました。</p>
    <p class="disclaimer">※効果には個人差があります</p>
  </div>
</div>

# 問題点:
# 1. 価格比較や「期間限定」といった有利誤認のリスクが高い項目を、HTMLに直書きさせている点は、
#    管理の放棄
# 2. ECサイトにおいて、二重価格表示（通常価格とセール価格）の妥当性を担保するのは、
#    HTMLのタグではなく「価格履歴のDBエビデンス」
# 3. 一定期間の販売実績がない価格を「通常価格」として表示させないロジックをシステムで組まずに、
#    「正しく表示せよ」と周知することに何の意味があるか
# 4. 「限定」「先着」といった表示を静的なテキストで周知する手法は、技術的敗北
# 5. 在庫が切れているのに「残りわずか」と表示され続ける、あるいは期限が過ぎても
#    「本日限定」と出続ける。これこそが景品表示法における「有利誤認」の典型
# 6. ※特別価格は2024年1月31日まで有効です とコードに埋め込むプロセスを、
#    数千の商品を扱う大規模サイトで維持できるとお考えか
# 7. キャンペーン期間が変わるたびにプルリクエストを送り、周知を回すのか
```

```typescript
// 価格履歴のDBエビデンスによる妥当性担保
class PriceHistoryValidator {
  async validatePriceDisplay(productId: string): Promise<PriceDisplay> {
    // 1. 価格履歴を取得
    const priceHistory = await this.priceHistoryRepository.getByProductId(productId);
    
    // 2. 一定期間（例: 30日）の販売実績を確認
    const recentSales = priceHistory.filter(price =>
      this.isWithinPeriod(price.date, 30) // 過去30日
    );
    
    // 3. 通常価格の妥当性を検証
    const regularPrice = await this.calculateRegularPrice(recentSales);
    
    // 4. セール価格の妥当性を検証
    const salePrice = await this.productRepository.getSalePrice(productId);
    
    if (!this.isValidPriceDisplay(regularPrice, salePrice, recentSales)) {
      throw new InvalidPriceDisplayError('価格表示が不適切です');
    }
    
    return {
      regularPrice,
      salePrice,
      evidence: {
        salesCount: recentSales.length,
        period: '30日間',
        priceHistory,
      },
    };
  }
  
  private isValidPriceDisplay(
    regularPrice: number,
    salePrice: number,
    recentSales: PriceHistory[]
  ): boolean {
    // 一定期間の販売実績がない価格を「通常価格」として表示させない
    if (recentSales.length === 0) {
      return false; // 販売実績がない場合は通常価格を表示できない
    }
    
    // 通常価格が実際に販売された価格であることを確認
    const hasRegularPriceSales = recentSales.some(sale =>
      sale.price === regularPrice
    );
    
    if (!hasRegularPriceSales) {
      return false; // 通常価格として表示する実績がない
    }
    
    return true;
  }
}
```

  async renderProductDisplay(productId: string): Promise<ProductDisplay> {
    // 1. 在庫APIから在庫数を取得
    const stock = await this.inventoryAPI.getStock(productId);
    
    // 2. スケジューラーからキャンペーン情報を取得
    const campaign = await this.scheduler.getCurrentCampaign(productId);
    
    // 3. ステートに基づいて表示を決定
    const display: ProductDisplay = {
      price: await this.getPrice(productId),
      stockStatus: this.calculateStockStatus(stock),
      campaignStatus: this.calculateCampaignStatus(campaign),
      limitedBadge: this.shouldShowLimitedBadge(stock, campaign),
    };
    
    return display;
  }
  
  private calculateStockStatus(stock: number): StockStatus {
    // 在庫が切れている場合は「残りわずか」を表示しない
    if (stock === 0) {
      return 'OUT_OF_STOCK';
    } else if (stock < 10) {
      return 'LOW_STOCK';
    } else {
      return 'IN_STOCK';
    }
  }
  
  private calculateCampaignStatus(campaign: Campaign | null): CampaignStatus {
    if (!campaign) {
      return 'NONE';
    }
    
    const now = new Date();
    
    // 期限が過ぎている場合は「本日限定」を表示しない
    if (now > campaign.endDate) {
      return 'EXPIRED';
    }
    
    return 'ACTIVE';

    if (stock === 0) {
      return false;
    }
    
    if (campaign && new Date() > campaign.endDate) {
      return false;
    }
    
    return true;
  }
}

interface DisclaimerMetadata {

  type: 'campaign_period' | 'stock_limit' | 'effect_disclaimer';
  content: string;
  legalChecked: boolean;
  effectiveFrom: Date;
  effectiveTo: Date;
}

class DisclaimerMetadataRepository {
  async getDisclaimer(
    productId: string,
    type: DisclaimerMetadata['type']
  ): Promise<DisclaimerMetadata> {
    // メタデータから取得
    const metadata = await this.metadataRepository.getByProductIdAndType(
      productId,
      type
    );
    
    // 法務チェック済みか確認
    if (!metadata.legalChecked) {
      throw new LegalCheckNotPassedError('法務チェックを通過していません');
    }
    
    // 有効期間内か確認
    const now = new Date();
    if (now < metadata.effectiveFrom || now > metadata.effectiveTo) {

    }
    
    return metadata;
  }
