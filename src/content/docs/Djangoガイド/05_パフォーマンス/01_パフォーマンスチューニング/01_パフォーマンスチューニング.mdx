# Djangoパフォーマンスチューニング完全ガイド

Djangoアプリケーションのパフォーマンスを最適化する方法を、実務で使える実装例とともに詳しく解説します。

## 1. パフォーマンスチューニングとは

### パフォーマンスの重要性

パフォーマンスは、ユーザー体験に直接影響します。適切な最適化により、アプリケーションの応答速度とスケーラビリティを向上させます。

```
パフォーマンス最適化の領域
   ├─ データベース最適化
   ├─ キャッシング
   ├─ クエリ最適化
   ├─ 静的ファイル最適化
   └─ サーバー設定最適化
```

## 2. データベースの最適化

### インデックスの活用

```python
# models.py
from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=200, db_index=True)  # インデックスを追加
    slug = models.SlugField(unique=True, db_index=True)  # ユニークインデックス
    created_at = models.DateTimeField(auto_now_add=True, db_index=True)  # 日付インデックス
    
    class Meta:
        indexes = [
            models.Index(fields=['title', 'created_at']),  # 複合インデックス
            models.Index(fields=['-created_at']),  # 降順インデックス
        ]
```

### N+1クエリ問題の解決

```python
# 問題のあるコード（N+1クエリ）
posts = Post.objects.all()
for post in posts:
    print(post.author.name)  # ループごとにクエリが発生

# 解決: select_related（一対一、一対多）
posts = Post.objects.select_related('author').all()
for post in posts:
    print(post.author.name)  # 追加クエリなし

# 解決: prefetch_related（多対多、逆方向の外部キー）
posts = Post.objects.prefetch_related('comments').all()
for post in posts:
    print(post.comments.all())  # 追加クエリなし
```

### 実務でのベストプラクティス

```python
# views.py
from django.views.generic import ListView
from .models import Post

class PostListView(ListView):
    model = Post
    
    def get_queryset(self):
        return Post.objects.select_related('author').prefetch_related(
            'comments', 'tags'
        ).filter(is_published=True)
```

## 3. QuerySetの最適化

### values()とvalues_list()

```python
# モデルインスタンス全体ではなく、必要なフィールドのみ取得
posts = Post.objects.values('title', 'author__name', 'created_at')

# 単一フィールドのリストとして取得
titles = Post.objects.values_list('title', flat=True)

# タプルのリストとして取得
post_data = Post.objects.values_list('id', 'title')
```

### only()とdefer()

```python
# 特定のフィールドのみをロード
posts = Post.objects.only('title', 'content', 'created_at')

# 特定のフィールドを遅延ロード
posts = Post.objects.defer('content')  # contentフィールドは必要になったときにロード
```

### count() vs len()

```python
# 高速なデータベースCOUNT
post_count = Post.objects.filter(is_published=True).count()

# 非推奨: 全データをロードしてからPythonでカウント
# post_count = len(Post.objects.filter(is_published=True))  # 非推奨
```

### exists()の使用

# 存在確認のみの場合
if Post.objects.filter(title='Test').exists():

## 4. キャッシング

### ビューレベルのキャッシング

```python
# views.py
from django.views.decorators.cache import cache_page
from django.utils.decorators import method_decorator

# 関数ベースビュー
@cache_page(60 * 15)  # 15分間キャッシュ
def post_list(request):
    posts = Post.objects.all()
    return render(request, 'posts/list.html', {'posts': posts})

# クラスベースビュー
@method_decorator(cache_page(60 * 15), name='dispatch')
class PostListView(ListView):
    model = Post
```

### テンプレートレベルのキャッシング

```html
<!-- テンプレート -->
{% load cache %}
{% cache 300 sidebar %}  <!-- 5分間キャッシュ -->
    <div class="sidebar">
        <!-- サイドバーのコンテンツ -->
    </div>
{% endcache %}
```

### 低レベルキャッシュAPI

```python
# views.py
from django.core.cache import cache

def expensive_view(request):
    cache_key = 'expensive_data'
    data = cache.get(cache_key)
    
    if data is None:
        # 重い処理
        data = expensive_operation()
        cache.set(cache_key, data, 300)  # 5分間キャッシュ
    
    return render(request, 'template.html', {'data': data})
```

### Celeryタスクの実装

```python
# tasks.py
from celery import shared_task
from django.core.mail import send_mail

@shared_task
def send_welcome_email(user_email):
    send_mail(
        'Welcome to MySite',
        'Thank you for signing up!',
        'from@example.com',
        [user_email],
        fail_silently=False,
    )

# views.py
from .tasks import send_welcome_email

def user_signup(request):
    user = User.objects.create(...)
    send_welcome_email.delay(user.email)  # 非同期で実行
    return redirect('success')
```

## 10. よくある問題と解決策

### 問題1: クエリが遅い

**原因:**
- インデックスが不足している
- N+1クエリが発生している

**解決策:**
```python
# インデックスの追加
class Post(models.Model):
    title = models.CharField(max_length=200, db_index=True)

# select_related/prefetch_relatedの使用
posts = Post.objects.select_related('author').prefetch_related('comments')
```

### 問題2: メモリ使用量が高い

**原因:**
- 大量のデータを一度に取得している
- 不要なフィールドをロードしている

**解決策:**
```python
# values()で必要なフィールドのみ取得
posts = Post.objects.values('title', 'created_at')

# イテレータの使用
for post in Post.objects.iterator():
    # 処理
    pass
```
