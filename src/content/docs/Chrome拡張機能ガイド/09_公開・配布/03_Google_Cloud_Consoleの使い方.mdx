## Google Cloud Consoleの使い方

Chrome拡張機能の公開と管理に使用するGoogle Cloud Console（Chrome Web Store Developer Dashboard）の使い方を説明します。

### Google Cloud Consoleとは

Google Cloud Console（Chrome Web Store Developer Dashboard）は、Chrome拡張機能をChrome Web Storeに公開・管理するためのダッシュボードです。

### アカウントの登録

#### 1. 初回登録

1. [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)にアクセス
2. Googleアカウントでログイン
3. 利用規約に同意
4. **$5の登録料を支払う**（初回のみ、一度支払えば以降は無料）

#### 2. 登録後の確認

- ダッシュボードにアクセスできる
- 「新しいアイテム」ボタンが表示される
- 拡張機能の管理が可能になる

### 拡張機能のアップロード

#### 1. ZIPファイルの準備

拡張機能をZIPファイルに圧縮します。

**含めるファイル:**
- manifest.json
- すべてのJavaScriptファイル
- すべてのHTMLファイル
- すべてのCSSファイル
- アイコンファイル
- その他のリソースファイル

**含めないファイル:**
- `.git`フォルダ
- `node_modules`フォルダ
- `.DS_Store`ファイル
- テストファイル
- ソースマップファイル（本番環境）

#### 2. アップロード手順

1. 「新しいアイテム」をクリック
2. ZIPファイルを選択してアップロード
3. アップロードが完了するまで待つ
4. 拡張機能の情報を入力

### 拡張機能情報の入力

#### 1. 基本情報

**必須項目:**
- **名前**: 拡張機能の名前（最大45文字）
- **説明**: 拡張機能の説明
  - **簡潔な説明**: 最大132文字（ストアの検索結果に表示）
  - **詳細な説明**: より詳しい説明（マークダウン対応）

**例:**

```
簡潔な説明:
ページのテキストを翻訳する拡張機能です。

詳細な説明:
# ページ翻訳拡張機能

選択したテキストを翻訳します。

## 機能
- 多言語対応
- 高速翻訳
- オフライン対応
```

#### 2. グラフィックアセット

**必須:**
- **アイコン**: 128x128ピクセル（PNG形式）

**推奨:**
- **スクリーンショット**: 
  - 1280x800ピクセル（推奨）
  - または640x400ピクセル
  - 1枚以上（最大5枚）

**オプション:**
- **プロモーション画像**: 440x280ピクセル
- **スモールタイル**: 440x280ピクセル

#### 3. カテゴリと言語

**カテゴリ:**
- アクセシビリティ
- ブログ
- 開発者ツール
- エンターテインメント
- 写真
- 生産性
- ショッピング
- ソーシャル
- その他

**言語:**
- 対応言語を選択
- 複数言語に対応可能

#### 4. 詳細情報

**プライバシーポリシー:**
- 個人情報を扱う場合は必須
- URLを入力

**ホームページURL:**
- 拡張機能のホームページ（オプション）

**サポートURL:**
- サポートページ（オプション）

### 審査プロセス

#### 1. 審査の流れ

1. **アップロード**: ZIPファイルをアップロード
2. **自動チェック**: 基本的なチェックが自動で実行される
   - manifest.jsonの検証
   - セキュリティチェック
   - ポリシー違反のチェック
3. **人間による審査**: スタッフによる審査（数日かかる場合がある）
4. **公開**: 審査が通ると公開される

#### 2. 審査時間

- **初回公開**: 通常1-3営業日
- **更新**: 通常1-2営業日
- **緊急更新**: 優先審査が可能（場合による）

#### 3. 審査状況の確認

ダッシュボードで審査状況を確認できます：

- **審査中**: 審査待ち
- **公開済み**: 公開されている
- **却下**: 審査に不合格

### バージョンの更新

#### 1. バージョン番号の更新

manifest.jsonのバージョン番号を更新：

```json
{
  "manifest_version": 3,
  "version": "1.0.1"  // 1.0.0から1.0.1に更新
}
```

#### 2. 更新のアップロード

1. ダッシュボードで拡張機能を選択
2. 「新しいバージョンをアップロード」をクリック
3. 更新されたZIPファイルをアップロード
4. 変更内容を説明

**変更内容の説明例:**

```
## バージョン1.0.1の変更内容

### 修正
- バグを修正
- パフォーマンスを改善

### 追加
- 新機能を追加
```

#### 3. 自動更新

Chromeは自動的に拡張機能を更新します：

- ユーザーは自動的に最新バージョンに更新される
- 更新通知は表示されない（通常）

### 統計情報の確認

#### 1. インストール数

- **総インストール数**: 累計のインストール数
- **アクティブユーザー数**: 現在アクティブなユーザー数
- **日次インストール数**: 1日あたりのインストール数

#### 2. 評価とレビュー

- **評価**: ユーザーからの評価（1-5星）
- **レビュー**: ユーザーからのレビュー
- **評価の傾向**: 評価の推移

#### 3. 国別統計

- 国別のインストール数
- 国別の評価

### レビューへの対応

#### 1. レビューの確認

1. ダッシュボードで拡張機能を選択
2. 「レビュー」タブをクリック
3. レビューを確認

#### 2. レビューへの返信

**良いレビューへの返信:**

```
ありがとうございます！今後も改善を続けます。
```

**問題のあるレビューへの対応:**

```
ご不便をおかけして申し訳ありません。
問題を修正しましたので、最新バージョンをご確認ください。
```

#### 3. 問題の修正

レビューで指摘された問題があれば、修正して更新：

1. 問題を確認
2. 修正を実装
3. バージョンを更新
4. レビューに返信

### 収益化（オプション）

#### 1. 有料拡張機能

- 拡張機能を有料で販売可能
- 価格を設定
- 決済はGoogleが処理

#### 2. 広告収入

- 拡張機能内に広告を表示
- 広告ネットワークと連携

### まとめ

Google Cloud Consoleの使い方：

1. **アカウント登録**: $5の登録料を支払う
2. **拡張機能のアップロード**: ZIPファイルをアップロード
3. **情報の入力**: 基本情報、グラフィックアセット、詳細情報
4. **審査**: 自動チェックと人間による審査
5. **公開**: 審査が通ると公開
6. **管理**: バージョン更新、統計確認、レビュー対応

Google Cloud Consoleを活用することで、Chrome拡張機能を効率的に公開・管理できます。
