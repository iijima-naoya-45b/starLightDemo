## Chrome Web Storeへの公開

Chrome拡張機能をChrome Web Storeに公開する手順を説明します。

### 公開前の準備

#### 1. 必要なもの

- **Googleアカウント**: Chrome Web Storeにアクセスするために必要
- **登録料**: 初回のみ$5（一度支払えば、以降は無料）
- **完成した拡張機能**: テスト済みの拡張機能

#### 2. 公開前のチェックリスト

- [ ] 拡張機能が正常に動作する
- [ ] エラーがない
- [ ] プライバシーポリシーがある（個人情報を扱う場合）
- [ ] アイコンが用意されている（16x16、48x48、128x128）
- [ ] スクリーンショットが用意されている（1280x800または640x400）
- [ ] 説明文が用意されている
- [ ] バージョン番号が設定されている

### パッケージ化

#### 1. ZIPファイルの作成

拡張機能のフォルダをZIPファイルに圧縮します。

**含めるファイル:**
- manifest.json
- すべてのJavaScriptファイル
- すべてのHTMLファイル
- すべてのCSSファイル
- アイコンファイル
- その他のリソースファイル

**含めないファイル:**
- `.git`フォルダ
- `node_modules`フォルダ
- `.DS_Store`ファイル
- テストファイル
- ソースマップファイル（本番環境）

#### 2. パッケージ化のコマンド

```bash
# ZIPファイルを作成（Mac/Linux）
zip -r my-extension.zip . -x "*.git*" -x "node_modules/*" -x "*.DS_Store"

# ZIPファイルを作成（Windows）
# エクスプローラーでフォルダを右クリック → 「送る」 → 「圧縮フォルダ」
```

### Chrome Web Store Developer Dashboard

#### 1. アカウントの登録

1. [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)にアクセス
2. Googleアカウントでログイン
3. 利用規約に同意
4. $5の登録料を支払う（初回のみ）

#### 2. 新しいアイテムの追加

1. 「新しいアイテム」をクリック
2. ZIPファイルをアップロード
3. 拡張機能の情報を入力

### 必要な情報の入力

#### 1. 基本情報

- **名前**: 拡張機能の名前（最大45文字）
- **説明**: 拡張機能の説明（最大132文字の簡潔な説明と、詳細な説明）
- **カテゴリ**: カテゴリの選択
- **言語**: 対応言語

#### 2. グラフィックアセット

- **アイコン**: 128x128ピクセル（必須）
- **スクリーンショット**: 1280x800または640x400ピクセル（1枚以上）
- **プロモーション画像**: 440x280ピクセル（オプション）
- **スモールタイル**: 440x280ピクセル（オプション）

#### 3. 詳細情報

- **プライバシーポリシー**: 個人情報を扱う場合は必須
- **ホームページURL**: 拡張機能のホームページ（オプション）
- **サポートURL**: サポートページ（オプション）

### 審査プロセス

#### 1. 審査の流れ

1. **アップロード**: ZIPファイルをアップロード
2. **自動チェック**: 基本的なチェックが自動で実行される
3. **人間による審査**: スタッフによる審査（数日かかる場合がある）
4. **公開**: 審査が通ると公開される

#### 2. 審査基準

- **機能性**: 拡張機能が正常に動作する
- **セキュリティ**: セキュリティ上の問題がない
- **コンテンツポリシー**: Chrome Web Storeのポリシーに準拠
- **ユーザー体験**: ユーザーにとって価値がある

#### 3. よくある却下理由

- **機能しない**: 拡張機能が正常に動作しない
- **セキュリティ問題**: セキュリティ上の問題がある
- **ポリシー違反**: Chrome Web Storeのポリシーに違反している
- **説明不足**: 説明が不十分

### 公開後の管理

#### 1. バージョンの更新

1. manifest.jsonのバージョン番号を更新
2. 変更をZIPファイルにパッケージ化
3. Developer Dashboardで「新しいバージョンをアップロード」
4. 変更内容を説明

#### 2. 統計情報の確認

- **インストール数**: インストールされた数
- **ユーザー数**: アクティブユーザー数
- **評価**: ユーザーからの評価
- **レビュー**: ユーザーからのレビュー

#### 3. レビューへの対応

- ユーザーからのレビューを確認
- 問題があれば修正
- 質問があれば回答

### まとめ

Chrome Web Storeへの公開：

1. **準備**: 拡張機能の完成、チェックリストの確認
2. **パッケージ化**: ZIPファイルの作成
3. **アップロード**: Developer Dashboardにアップロード
4. **情報入力**: 基本情報、グラフィックアセット、詳細情報
5. **審査**: 自動チェックと人間による審査
6. **公開**: 審査が通ると公開
7. **管理**: バージョン更新、統計確認、レビュー対応

適切に準備することで、スムーズにChrome Web Storeに公開できます。
